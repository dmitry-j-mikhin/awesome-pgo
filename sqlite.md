# SQLite: PGO results

Here I want to share my results with PGO SQLite optimization.

## Test environment

* Fedora 38
* Linux kernel 6.3.7
* AMD Ryzen 9 5900x
* 48 Gib RAM
* SSD Samsung 980 Pro 2 Tib
* Clang 16 (from the Fedora repositories). I use Clang just because I prefer LLVM-based tooling
* SQLite version: latest commit (`40ee72947e39f54` commit)

## Tested configurations

I have tested the following SQLite configurations (with corresponding `CFLAGS` and `CXXFLAGS`):

* Release: `CC=clang CFLAGS="-O3" ../sqlite/configure`
* Release with PGO: `CC=clang CFLAGS="-O3 -fprofile-instr-use=code.profdata" ../sqlite/configure`

As a PGO technique I use `-fprofile-instr-generate`/`-fprofile-instr-use` options from Clang. Build instrumented SQLite version, run ClickBench with the instrumented SQLite, collect instrumentation data, then rebuild SQLite again with the collected data.

## Benchmark

I use [ClickBench](https://github.com/ClickHouse/ClickBench/tree/main/sqlite) as a SQLite benchmark. I am quite familiar with this bench quite already and it's proven by multiple people.

## Results

Here are results of running ClickBench test suite on different configurations. All configurations are benchmarked on the same machine, with the same SQLite configuration, multiple times, etc. Results are shown in `time` format (per query). Each query is repeated 3 times to get a more precise time measure. The most interesting time to check is `user` time.
 
Also, I shared results for Instrumentation mode so you would be able to estimate how slow SQLite is in Instrumentation mode for PGO.
